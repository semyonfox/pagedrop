# PageDrop

Publish an HTML page. Get a shareable URL.

PageDrop is a small self-hosted service for static reports, dashboards, demos,
documentation, and pages generated by AI agents. It accepts a standalone HTML
file, ZIP archive, or local directory and returns a cryptographically random
public link.

```console
$ pagedrop upload --expires 7d ./report
Published: https://pages.example.com/p/4WcJXG7i0Jo8F2K_48VcmQ/
ID: 4WcJXG7i0Jo8F2K_48VcmQ
Expires: 2026-07-29T20:00:00Z
```

PageDrop deliberately runs no uploaded server-side code. HTML, CSS, JavaScript,
images, fonts, JSON, SVG, WASM, and other static assets are served as files.

## Features

- One lightweight Go binary for both server and CLI
- Standalone HTML, ZIP, and automatic directory uploads
- 128-bit URL-safe random page identifiers
- Bearer-token protected management API
- SQLite metadata and filesystem content storage
- Configurable expiry with automatic cleanup
- Atomic page replacement without changing the URL
- Safe ZIP extraction and compressed/extracted/file-count limits
- ETags and cache revalidation without stale replacements
- Structured JSON API and deterministic CLI output modes
- Non-root, capability-free Docker deployment
- Optional Cloudflare Tunnel sidecar

## Quick start

```bash
git clone https://github.com/semyonfox/pagedrop.git
cd pagedrop
cp .env.example .env
openssl rand -hex 32
```

Put the generated value in `.env` as `PAGEDROP_TOKEN`, then run:

```bash
docker compose up --build -d
curl http://127.0.0.1:8788/health
```

Build and configure the CLI:

```bash
go build -o pagedrop ./cmd/pagedrop
./pagedrop configure \
  --server http://127.0.0.1:8788 \
  --token YOUR_GENERATED_TOKEN
```

The configuration is stored with mode `0600` at
`~/.config/pagedrop/config.toml` on Linux.

## Use

```bash
# A self-contained file
pagedrop upload report.html

# A complete site; the CLI creates the ZIP automatically
pagedrop upload --title "Benchmark" --expires 7d ./report-site

# Machine-readable output
pagedrop upload --json ./report-site

# Print only the resulting URL (ideal for agents)
pagedrop upload --quiet ./report-site

# Manage pages
pagedrop list
pagedrop info PAGE_ID
pagedrop replace PAGE_ID ./updated-report
pagedrop delete PAGE_ID
```

Directories and ZIP archives must contain `index.html` at their root. Use
relative asset URLs such as `assets/chart.js`; root-relative URLs such as
`/assets/chart.js` refer to the PageDrop host root and will not work beneath a
page URL.

Supported expiration values include `1h`, `7d`, `30d`, and `never`.

## AI-agent instruction

```text
Create a static website in a temporary directory with index.html at its root.
Publish it by running:

    pagedrop upload --expires 7d --quiet DIRECTORY

Return the printed URL to the user. Never place credentials or private data in
the generated page.
```

## Cloudflare Tunnel

PageDrop binds host port `8788` to loopback only. The optional tunnel container
reaches the service through Docker's private network, so no router port-forward
or LAN-wide listener is needed.

1. Create a remotely managed Cloudflare Tunnel.
2. Add a public hostname such as `pages.example.com`.
3. Set its service URL to `http://pagedrop:8080`.
4. Put the connector token and hostname in `.env`:

```env
TUNNEL_TOKEN=YOUR_CLOUDFLARE_TUNNEL_TOKEN
PAGEDROP_PUBLIC_BASE_URL=https://pages.example.com
```

5. Start both services:

```bash
docker compose --profile tunnel up --build -d
```

Uploaded pages are arbitrary untrusted JavaScript. Prefer a dedicated content
hostname or registrable domain with no authentication cookies. PageDrop itself
uses bearer headers and does not set cookies.

## HTTP API

All management routes require `Authorization: Bearer TOKEN`.

| Method | Route | Purpose |
| --- | --- | --- |
| `POST` | `/api/v1/pages` | Create a page |
| `GET` | `/api/v1/pages` | List pages |
| `GET` | `/api/v1/pages/{id}` | Get metadata |
| `PUT` | `/api/v1/pages/{id}/content` | Replace content atomically |
| `DELETE` | `/api/v1/pages/{id}` | Delete a page idempotently |
| `GET` | `/health` | Health and database status |
| `GET` | `/p/{id}/...` | Public page content |

Uploads use `multipart/form-data` with `file` plus optional `title` and
`expires_in` fields.

```bash
curl -H "Authorization: Bearer $PAGEDROP_TOKEN" \
  -F "file=@site.zip" \
  -F "title=Network benchmark" \
  -F "expires_in=7d" \
  http://127.0.0.1:8788/api/v1/pages
```

Errors are structured:

```json
{"error":{"code":"UNSAFE_ARCHIVE","message":"Archive contains a traversal path."}}
```

## Storage and caching

SQLite stores metadata only. Page files use versioned directories:

```text
data/
├── pagedrop.db
└── pages/
    └── PAGE_ID/
        └── v2/
            ├── index.html
            └── assets/
```

Replacement fully validates and activates a new version before updating the
database pointer. Public responses include version-dependent ETags and
`Cache-Control: public, no-cache`: browsers and CDNs may retain bytes but must
revalidate, avoiding stale assets without giving up cheap `304 Not Modified`
responses. A separate cache/database is unnecessary at this scale.

Back up the entire `data` directory consistently. Stop the service first for
the simplest snapshot, or use SQLite's backup facilities before copying live
data.

## Configuration

| Variable | Default |
| --- | --- |
| `PAGEDROP_TOKEN` | required, minimum 32 characters |
| `PAGEDROP_LISTEN_ADDR` | `:8080` |
| `PAGEDROP_DATA_DIR` | `./data` |
| `PAGEDROP_PUBLIC_BASE_URL` | `http://localhost:8080` |
| `PAGEDROP_DEFAULT_EXPIRY` | `30d` |
| `PAGEDROP_MAX_EXPIRY` | `365d` |
| `PAGEDROP_MAX_UPLOAD_BYTES` | `26214400` (25 MiB) |
| `PAGEDROP_MAX_EXTRACTED_BYTES` | `104857600` (100 MiB) |
| `PAGEDROP_MAX_FILES` | `1000` |

## Development

Requires Go 1.26 or newer.

```bash
make check
make docker
```

Tests cover authentication, HTML publication, ZIP assets, traversal rejection,
expiry, cache validation, atomic replacement, deletion, and post-delete access.

## Scope

PageDrop v1 is intentionally single-owner. It does not provide registration,
teams, billing, server-side runtimes, passwords for individual pages, analytics,
or object storage. SQLite plus local files is the simplest reliable deployment;
PostgreSQL and R2/S3 become useful only when running multiple server replicas.
